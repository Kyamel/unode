<Aside type="caution" title="Alpha">
Unode is in **alpha**. The protocol, ABI, and runtime boundaries are still
stabilizing and may change between releases. It is ready to build and
experiment with, not yet to depend on in production.
</Aside>

## What is Unode?

Unode is an experimental UI framework for **plugin-first** applications. A
plugin describes a screen once as serializable *semantic intent* — a heading, a
danger action, a cover image — never pixels or colors. Plugins compile to
WebAssembly and run inside a sandbox. Environment-specific runtimes then render
that intent in its right native shape: DOM on the web, terminal cells in a TUI.

The core protocol is written in Rust. The wire format is JSON/IR, so Unode can
be embedded in applications built with React, Svelte, Vue, or any other stack.

<CardGrid stagger>
	<Card title="Intent, not pixels" icon="pencil">
		Plugins declare *what* something is. The renderer decides how it looks
		on each host. One screen definition, many native renderings.
	</Card>
	<Card title="Sandboxed by default" icon="approve-check">
		Plugins run as WASM modules with isolated linear memory. They can only
		touch the outside world through explicit, permission-gated host calls.
	</Card>
	<Card title="Granular reactivity" icon="random">
		Host-owned state drives Solid-style targeted updates. A state write
		patches only the bound nodes — `render()` is never called again.
	</Card>
	<Card title="Framework-agnostic" icon="puzzle">
		React and Svelte are maintained adapters over the same IR and patch
		ops. Add a Vue or custom adapter without touching core semantics.
	</Card>
</CardGrid>
